Git Specifying Key

• SSH used -i /path/to/key to ssh to a remote server using specific key file,
git does not have -i option
• In Git there steps need to be done to accomplish this:
   ◦ Add this to ~/.ssh/config file (create if not existing)
     Host <RemoteServer> # eg. GitLab
      HostName <remote-server-url> # eg. gitlab.com
      User git
      IdentityFile ~/.ssh/<key> # eg. ironaire
      IdentitiesOnly yes
   ◦ If git fails still, run ssh-add ~/.ssh/<key> to add the identity to local ssh.
   ◦ Optionally run ‘ssh <RemoteServer>’ to test the connection
   ◦ Add remote server to git project
     git remote add origin <RemoteServer>:path/to/repository.git
   ◦ Do a push to test
     git push
