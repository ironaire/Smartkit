[ ] Fix gitlab ci 'fatal: unable to access 'https://gitlab-ci-token:xxxxxxxxxxxxxxxxxxxx@code.bmi.osumc.edu/ird/my-experience.git/': The requested URL returned error: 500'
    This is mostly because apache is sitting in front of gitlab server.
    gitlab instance has 2 http web servers, one for gitlab web ui, and another for git relative calls(clone through git
    and http/https).

    To fix the issue, we could try these:
    [ ] update /etc/gitlab/gitlab.rb with these 2 lines:
        gitlab_git_http_server['listen_network'] = 'tcp'
        gitlab_git_http_server['listen_addr'] = '8282'

    [ ] update /etc/http/conf.d/gitlab.conf with the rewrite:
        RewriteRule /[-\/\w\.]+\.git\/ http://127.0.0.1:8282%{REQUEST_URI} [P,QSA,L]
        RewriteRule .* http://127.0.0.1:8080%{REQUEST_URI} [P,QSA]
